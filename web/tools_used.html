~ First of all ~

The complexity of the AlvoHive project lies in its infrastructure using Apache for web hosting, PostgreSQL for the database, and Docker for containerization. Here is an exhaustive list of what was used to create AlvoHive:


Virtual Machines and Tools

Kali Linux: Deployed in a virtual machine environment for conducting penetration tests.

Target VM: Vulnerable systems such as Metasploitable 2 are used as target virtual machines.


Penetration Testing Tools

Nmap: Utilized for conducting comprehensive network scans to discover hosts and services on a computer network.

Metasploit: A powerful tool used for exploitation and vulnerability testing, allowing for the simulation of attacks to find security weaknesses.


Development Stack

Web Development

HTML: Used for structuring web content.

CSS: Employed for styling web pages to enhance their visual appeal.

JavaScript: Adds interactive features and functionalities to web pages.


Backend Development

Python: Utilized for scripting, data analysis, and backend development.

Flask: A web framework used for building REST APIs.

SQLAlchemy: An ORM (Object Relational Mapper) for database management, providing a way to interact with the database using Python objects.


Database

SQLite: Used for storing penetration test results and other data in a lightweight database.

PostgreSQL: Employed for handling more complex queries and data management needs.


Docker

Docker: A platform for developing, shipping, and running applications in containers.

Docker Compose: A tool for defining and running multi-container Docker applications, facilitating easy orchestration of services.


APIs

CVE Search API: Used for fetching CVE (Common Vulnerabilities and Exposures) descriptions and statuses.

REST API: Allows for interfacing with the web application and fetching data.


Additional Tools

Pandoc: A universal document converter used for converting Markdown files to PDF reports.

mdutils: A Python package for generating and formatting Markdown files.


Risk Mitigation and Testing

Unit Testing: Using frameworks like PyUnit (Unittest) for testing individual components and modules to ensure code reliability.


Version Control: Git and GitHub are employed for versioning and managing the project repository.


To see more about the infrastructure carrying this whole project, check out the document detailing:

  • Creation of directories and Docker build files: Apache Dockerfile setup with an Ubuntu base image, Apache installation, SSL configuration, and port exposure.

  • Configuring docker-compose.yml: Services for Apache and Postgres, with environment settings, ports, volumes, and network configurations.

  • Starting containers: Commands for starting, running in the background, checking status, and accessing containers.

  • Apache and Postgres configurations: Customizing Apache and Postgres settings, mapping config files, and container restarts.

  • Backup and restore processes: Commands for creating and restoring backups.

  • Connection tests: Methods to test Apache and Postgres connections, including Python scripts and database client setups.
